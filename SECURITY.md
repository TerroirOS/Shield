# Security Policy

## Reporting a Vulnerability

Please do not disclose security issues publicly before coordinated review.

Report vulnerabilities by opening a private security advisory in GitHub for this repository or by contacting project maintainers directly.

Include:

- Description of the issue
- Potential impact
- Reproduction steps
- Affected files/specs

## Scope Note

This repository currently contains documentation and specifications, not production runtime services.
Security findings can still apply to:

- API design risks
- Data model exposure risks
- Governance control gaps
- Unsafe example patterns

## Handling Policy

- We acknowledge reports as quickly as possible.
- We validate severity and expected remediation path.
- We publish fixes and relevant notes after remediation.
